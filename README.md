# Customer Orders Analysis

This project uses a simulated dataset to practice SQL-based data analysis on customer transactions.

## Dataset
The dataset consists of two tables:
- customers: customer information including city and signup year
- orders: transaction data including order amount and date

## Objectives
- Calculate total revenue
- Identify top customers by spending
- Analyze revenue by city
- Evaluate customer order frequency
- Compare revenue by signup cohort

## Tools Used
- SQL (SQLite)
- DB Browser for SQLite

## Key Queries
The analysis includes:
- GROUP BY aggregations (SUM, COUNT, AVG)
- JOIN operations between customers and orders
- Sorting and filtering results using ORDER BY

## Key Insights
- Total revenue across all customers is 300
- Customer 101 generated the highest total revenue (80)
- Customer 103 made the largest single purchase (100)
- New York generated the highest total revenue
- Customers who signed up in 2023 generated more revenue than those in 2024
- Average order value is 50

- SQL querying
- Data aggregation and analysis
- Relational data understanding
